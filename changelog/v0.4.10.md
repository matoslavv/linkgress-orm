## Fix `jsonbConditionUnwrap` type signature

Changed `jsonbConditionUnwrap` to use a generic signature `<T>(value: T): T` so it preserves the input type for correct generic inference in condition functions like `eq`. Previously returned `any`/`string` which broke type inference when used with typed `JsonbElement` properties.

```typescript
// Now works without type errors:
jsonbArraySome<IntegrationConfig>(p.integrationConfig, c =>
  eq(c.type, jsonbConditionUnwrap(IntegrationType.VILLAPRO))
)
```

### Files Changed

- `src/query/conditions.ts` — `jsonbConditionUnwrap` signature changed to `<T>(value: T): T`
