## `jsonbArraySome` & `SqlFragment` in `FieldLike`

### `jsonbArraySome` — query JSONB arrays with `.some()` semantics

New helper to check if any element in a JSONB array column matches a predicate. Generates an `EXISTS` subquery with `jsonb_array_elements`.

```typescript
db.products.where(p =>
  jsonbArraySome<IntegrationConfig>(p.integrationConfig, c =>
    and(
      eq(c.type, IntegrationType.VILLAPRO),
      isNotNull(c.config.villaproSystToken)
    )
  )
)
// → WHERE EXISTS (SELECT 1 FROM jsonb_array_elements("product"."integration_config") AS __elem
//     WHERE (__elem->>'type' = $1 AND __elem->'config'->>'villaproSystToken' IS NOT NULL))
```

The callback receives a `JsonbElement<T>` — a fully typed proxy that builds JSONB path expressions (`->` for intermediate, `->>` for leaf text extraction). Property types flow through correctly (e.g. enum fields accept enum values in `eq()`). Works with all existing condition functions (`eq`, `ne`, `like`, `isNotNull`, etc.) and supports deep nesting (`c.config.villaproSystToken`).

### `SqlFragment` added to `FieldLike` type

`sql` tagged templates can now be used as the field (left-hand side) in condition functions without `as any`:

```typescript
// Before: like(sql`${p.internalId}::varchar(255)` as any, '%search%')
// After:  like(sql`${p.internalId}::varchar(255)`, '%search%')
```
