## New Features

### `getColumnKeys()` - `includePrimaryKey` Option

Added `includePrimaryKey` option to `getColumnKeys()` method. This allows excluding primary key columns from the returned keys, which is useful for building dynamic update objects or forms.

```typescript
// Get all column keys (default - includes primary key)
const allKeys = db.users.getColumnKeys();
// Returns: ['id', 'username', 'email', 'age', 'isActive', ...]

// Exclude primary key columns (useful for updates)
const updateableKeys = db.users.getColumnKeys({ includePrimaryKey: false });
// Returns: ['username', 'email', 'age', 'isActive', ...] (without 'id')

// Combine with includeNavigation
const nonPkWithNav = db.users.getColumnKeys({
  includePrimaryKey: false,
  includeNavigation: true
});
// Returns: ['username', 'email', 'age', 'isActive', 'posts', 'orders', ...]
```

### Full Change List
- Added `includePrimaryKey` option to `getColumnKeys()` method (defaults to `true` for backwards compatibility)
- Added 12 tests for `getColumnKeys()` covering all option combinations
