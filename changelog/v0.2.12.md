## New Features

### UNION and UNION ALL Support

Added support for combining multiple SELECT queries using SQL UNION operations.

- `union()` - Combines queries and removes duplicate rows
- `unionAll()` - Combines queries and keeps all rows including duplicates (faster)

```typescript
// Basic UNION (removes duplicates)
const result = await db.users
  .select(u => ({ id: u.id, name: u.username }))
  .union(db.customers.select(c => ({ id: c.id, name: c.name })))
  .toList();

// UNION ALL (keeps duplicates - faster)
const allLogs = await db.errorLogs
  .select(l => ({ timestamp: l.createdAt, message: l.message }))
  .unionAll(db.infoLogs.select(l => ({ timestamp: l.createdAt, message: l.message })))
  .toList();

// Multiple unions with ordering and pagination
const sorted = await query1
  .union(query2)
  .unionAll(query3)
  .orderBy(r => r.name)
  .limit(100)
  .offset(20)
  .toList();

// Union queries from different tables with compatible columns
const names = await db.users
  .select(u => ({ name: u.username }))
  .union(db.posts.select(p => ({ name: p.title })))
  .orderBy(r => r.name)
  .toList();
```

### UnionQueryBuilder Methods

- `union(query)` - Add a query with UNION
- `unionAll(query)` - Add a query with UNION ALL
- `orderBy(selector)` - Order the combined result
- `limit(count)` - Limit results
- `offset(count)` - Skip results
- `toList()` - Execute and return results
- `firstOrDefault()` - Get first result or null
- `count()` - Count combined results
- `toSql()` - Get generated SQL for debugging

### New Exports

- `UnionQueryBuilder` - Class for building UNION queries
- `UnionType` - Type definition (`'UNION' | 'UNION ALL'`)
- `isUnionQueryBuilder()` - Type guard function

### Future Queries (NHibernate-style Batch Execution)

Added support for deferred query execution using Future queries. This pattern allows you to queue multiple queries and execute them together, with proper type inference for results.

- `future()` - Creates a deferred query that returns an array of results
- `futureFirstOrDefault()` - Creates a deferred query that returns a single result or null
- `futureCount()` - Creates a deferred query that returns a count

```typescript
// Create future queries
const q1 = db.users.select(u => ({ id: u.id, name: u.username })).future();
const q2 = db.posts.where(p => eq(p.authorId, 1)).select(p => p).futureFirstOrDefault();
const q3 = db.comments.futureCount();

// Execute all at once with properly typed results
const [users, firstPost, commentCount] = await FutureQueryRunner.runAsync([q1, q2, q3] as const);
// users: { id: number, name: string }[]
// firstPost: Post | null
// commentCount: number
```

#### Individual Execution

Future queries can also be executed individually:

```typescript
const futureQuery = db.users
  .select(u => ({ name: u.username }))
  .orderBy(u => u.name)
  .limit(10)
  .future();

// Execute when ready
const users = await futureQuery.execute();

// Get the SQL for debugging
const sql = futureQuery.getSql();
```

#### FutureQueryRunner Methods

- `runAsync(futures)` - Execute multiple future queries (use `as const` for proper type inference)

#### New Exports

- `FutureQuery<T>` - Class for array result queries
- `FutureSingleQuery<T>` - Class for single result queries
- `FutureCountQuery` - Class for count queries
- `FutureQueryRunner` - Static runner for batch execution
- `AnyFutureQuery` - Union type of all future query types
- `FutureQueryResult<T>` - Type helper to extract result type
- `FutureQueryResults<T>` - Type helper for tuple results
- `isFutureQuery()` - Type guard function
- `isFutureSingleQuery()` - Type guard function
- `isFutureCountQuery()` - Type guard function
