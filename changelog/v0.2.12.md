## New Features

### UNION and UNION ALL Support

Added support for combining multiple SELECT queries using SQL UNION operations.

- `union()` - Combines queries and removes duplicate rows
- `unionAll()` - Combines queries and keeps all rows including duplicates (faster)

```typescript
// Basic UNION (removes duplicates)
const result = await db.users
  .select(u => ({ id: u.id, name: u.username }))
  .union(db.customers.select(c => ({ id: c.id, name: c.name })))
  .toList();

// UNION ALL (keeps duplicates - faster)
const allLogs = await db.errorLogs
  .select(l => ({ timestamp: l.createdAt, message: l.message }))
  .unionAll(db.infoLogs.select(l => ({ timestamp: l.createdAt, message: l.message })))
  .toList();

// Multiple unions with ordering and pagination
const sorted = await query1
  .union(query2)
  .unionAll(query3)
  .orderBy(r => r.name)
  .limit(100)
  .offset(20)
  .toList();

// Union queries from different tables with compatible columns
const names = await db.users
  .select(u => ({ name: u.username }))
  .union(db.posts.select(p => ({ name: p.title })))
  .orderBy(r => r.name)
  .toList();
```

### UnionQueryBuilder Methods

- `union(query)` - Add a query with UNION
- `unionAll(query)` - Add a query with UNION ALL
- `orderBy(selector)` - Order the combined result
- `limit(count)` - Limit results
- `offset(count)` - Skip results
- `toList()` - Execute and return results
- `firstOrDefault()` - Get first result or null
- `count()` - Count combined results
- `toSql()` - Get generated SQL for debugging

### New Exports

- `UnionQueryBuilder` - Class for building UNION queries
- `UnionType` - Type definition (`'UNION' | 'UNION ALL'`)
- `isUnionQueryBuilder()` - Type guard function
