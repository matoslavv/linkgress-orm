## Feature: `countOver()` for single-query pagination

### What

New `countOver()` method on query builders and `DbEntityTable` that returns both the paginated data and the total matching row count in a single database query using PostgreSQL's `COUNT(*) OVER()` window function.

### Why

Paginated APIs typically require two separate queries: one for the data page (`toList()` with `limit`/`offset`) and one for the total count (`count()`). On large tables this doubles the database load. `countOver()` combines both into a single query, cutting the round-trips in half.

### Usage

```typescript
const { data, totalCount } = await db.users
  .where(u => eq(u.isActive, true))
  .orderBy(u => u.username)
  .limit(50)
  .offset(0)
  .countOver();

// data: the 50 users for this page
// totalCount: total number of active users (before LIMIT/OFFSET)
```
