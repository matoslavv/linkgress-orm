## New Features

### Navigation Property Support in `RETURNING` Statements

Added support for navigation properties in `RETURNING` clauses for all mutation operations (`delete()`, `update()`, `insert()`, `insertBulk()`, `upsertBulk()`). This allows you to return data from related tables in a single query.

```typescript
// Delete and return navigation property data
const deleted = await db.orders
  .where(o => eq(o.id, orderId))
  .delete()
  .returning(o => ({
    orderId: o.id,
    orderStatus: o.status,
    userName: o.user!.username,  // Navigation property!
    userEmail: o.user!.email,
  }));

// Update and return navigation property data
const updated = await db.posts
  .where(p => eq(p.id, postId))
  .update({ title: 'New Title' })
  .returning(p => ({
    postId: p.id,
    postTitle: p.title,
    authorName: p.user!.username,  // Navigation property!
  }));

// Insert and return navigation property data
const inserted = await db.posts
  .insert({ title: 'New Post', userId: userId, views: 0 })
  .returning(p => ({
    postId: p.id,
    postTitle: p.title,
    authorName: p.user!.username,  // Navigation property!
  }));
```

When navigation properties are detected in the `RETURNING` selector, the ORM automatically uses a CTE-based query to fetch related data. When no navigation properties are used, the simple `RETURNING` clause is used (no extra complexity).

## Bug Fixes

### `delete()` with Navigation Properties in WHERE
Fixed `delete()` not generating proper JOINs for navigation properties used in WHERE clauses. PostgreSQL DELETE requires `USING` clause for JOINs instead of standard JOIN syntax.

```typescript
// Before: Crashed with "missing FROM-clause entry for table"
// After: Correctly uses USING clause to join related tables
await db.cartItems
  .where(p => and(
    eq(p.cart.uuid, cartId),
    flagHas(p.product.mode, ProductMode.TARIFF)
  ))
  .delete();
```

Also fixed:
- Multi-level navigation in `delete()` (e.g., `ot.task!.level!.name`)
- `RETURNING` clause column ambiguity when using navigation properties
- Subquery conditions combined with navigation properties in `delete()`

### Full Change List
- Added `detectNavigationInReturning()` method to detect navigation properties in RETURNING selectors
- Added `buildReturningWithNavigation()` method to build CTE-based queries when navigation is detected
- Added `mapReturningResultsWithNavigation()` method to properly apply type mappers for navigation fields
- Updated `delete()`, `update()`, `insert()`, `insertBulk()`, `upsertBulk()` to support navigation in RETURNING
- Fixed `delete()` to detect navigation property JOINs and use PostgreSQL `USING` clause
- Fixed `RETURNING` clause to qualify columns with table name when using `USING` to avoid ambiguity
- Added comprehensive tests for `delete()` with navigation properties and subqueries
- Added comprehensive tests for navigation properties in RETURNING (14 tests)
