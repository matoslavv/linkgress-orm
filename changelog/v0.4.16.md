## Feature: `exists()` and `notExists()` function syntax for collection navigation

### What

The `exists()` and `notExists()` helper functions now accept collection navigation properties directly, in addition to `Subquery` objects. This enables a cleaner, more readable syntax for filtering by related entities.

### Why

Previously, checking existence on a collection navigation required either:
- The method form: `p.postComments!.where(...).exists()` 
- The verbose subquery form: `inSubquery(p.id, db.table.where(...).select(...).asSubquery())`

The function form `exists(p.collection!.where(...))` reads more naturally and mirrors how developers think about the query: "where exists related items matching condition."

### Usage

```typescript
// Function form (new)
db.posts.where(p => exists(
  p.postComments!.where(pc => eq(pc.comment, 'Great post!'))
));

// NOT EXISTS
db.posts.where(p => notExists(
  p.postComments!.where(pc => eq(pc.comment, 'spam'))
));

// With multiple conditions (mirrors real-world filtering by relation)
db.users.where(u => exists(
  u.orders!.where(o => and(
    eq(o.status, 'pending'),
    eq(o.userId, masterUserId)
  ))
));

// Nested navigation still works
db.posts.where(p => exists(
  p.postComments!.where(pc =>
    pc.order!.orderTasks!.where(ot => eq(ot.sortOrder, 1)).exists()
  )
));

// Method form still works unchanged
db.posts.where(p => p.postComments!.where(...).exists());
```

### Details

- `exists()` and `notExists()` accept `Subquery | CollectionSubquerySource` — a duck-typed interface requiring only `exists(): SqlFragment<boolean>`
- `CollectionSubquerySource` interface is exported for consumer use
- `ExistsConditionBase` abstract base class extracts shared constructor, `resolved` field, `getFieldRefs()`, and `buildSubquerySql()` — `ExistsCondition` and `NotExistsCondition` only override `buildSql()`
- Generic design: the `CollectionSubquerySource` interface can be reused by future wrapper functions

### Files Changed

- `src/query/subquery.ts` — added `CollectionSubquerySource` interface and `ExistsConditionBase` base class; `ExistsCondition` and `NotExistsCondition` extend the base and accept `Subquery | CollectionSubquerySource`
- `src/index.ts` — exported `CollectionSubquerySource` type
- `tests/queries/exists-function-collection.test.ts` — 7 test cases covering function form in WHERE/SELECT, equivalence with method form, notExists, nested navigation, and multi-condition filtering
